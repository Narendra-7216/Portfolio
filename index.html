Distributed Rate Limiter (Systems Project)

Designed and implemented a distributed rate limiter to enforce per-user API request limits across horizontally scaled services. Implemented a token-bucket algorithm using Redis Lua scripts to ensure atomic request handling under high concurrency. Built the service stateless for seamless horizontal scaling, optimized request handling to O(1) supporting 10K+ req/s with sub-5ms latency, and implemented TTL-based cleanup to prevent stale state. Analyzed failure scenarios including Redis outages and burst traffic; documented tradeoffs between fail-open and fail-closed behavior.
